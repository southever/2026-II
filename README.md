# 2026-工科创II
最终汇报执行计划
一、总目标
本次最终汇报以 6 月 15 日（周一）前完成全部材料封版 为目标。在已有中期成果基础上，我们需要把项目从“完成基础仿真”提升为“考虑真实约束的家庭光伏-储能-EV 系统工程评估”。
最终成果包括：
最终项目报告
数据文件
MATLAB 代码
答辩 PPT
最终结论不写成“某个容量或策略绝对最优”，而采用条件化推荐：在不同电价机制、储能成本、EV 充电行为和运行目标下，分别给出经济型、均衡型、鲁棒型方案。
二、总体依赖关系
项目推进顺序为：
A 统一口径 → B 负荷/PV 数据 → C EV 负荷与总负荷 → D 储能选型与敏感性分析 → E 策略优化 → A 汇总报告/PPT
依赖说明：
B 的基础负荷和光伏数据，是 C、D、E 的输入。
C 的 EV 负荷和总负荷，是 D 储能扫描和 E 策略优化的输入。
D 的储能候选方案，是 E 做策略对比的基础。
A 可以提前搭建报告和 PPT 框架，但最终结论必须等 B/C/D/E 的结果出来后再封版。

六、关键风险控制
B/C 延迟会影响 D：B/C 周五下午先交第一版数据，D 可先跑框架，晚上替换最终数据。
D/E 结论可能冲突：D 只给三类候选方案，E 说明不同策略下的适用条件。
E 任务过重：已将 EV 协同交给 C，敏感性分析交给 D，多目标评分由 A 汇总，E 专注任务六。
电价预测做不完：最低交付为电价情景分析，不强行做复杂预测算法。
报告和 PPT 数字不一致：A 只采用各成员最终汇总表中的数字。
模型被质疑过简化：主动说明模型假设，并用成本、寿命、EV 行为、电价情景和敏感性分析证明扩展价值。
