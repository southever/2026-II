# 最终汇报执行计划

## 一、总目标

本次最终汇报以 **周一前完成全部材料封版** 为目标。在已有中期成果基础上，把项目从“完成基础仿真”提升为“考虑真实约束的家庭光伏-储能-EV 系统工程评估”。

最终成果包括：

- 最终项目报告
- 数据文件
- MATLAB 代码
- 答辩 PPT

最终结论不写成“某个容量或策略绝对最优”，而采用条件化推荐：

> 在不同电价机制、储能成本、EV 充电行为和运行目标下，分别给出经济型、均衡型、鲁棒型方案。

---

## 二、总体依赖关系

项目推进顺序为：

**A 统一口径 → B 负荷/PV 数据 → C EV 负荷与总负荷 → D 储能选型与敏感性分析 → E 策略优化 → A 汇总报告/PPT**

依赖说明：

- B 的基础负荷和光伏数据，是 C、D、E 的输入。
- C 的 EV 负荷和总负荷，是 D 储能扫描和 E 策略优化的输入。
- D 的储能候选方案，是 E 做策略对比的基础。
- A 可以提前搭建报告和 PPT 框架，但最终结论必须等 B/C/D/E 的结果出来后再封版。

## 成员 D：储能选型、成本、寿命与敏感性分析负责人

**核心目标：**  
把任务五从“扫描最优值”升级为“工程选型”。

**负责内容：**

- 复核储能容量/功率双维度扫描。
- 明确并检查储能约束：
  - SOC 上限
  - SOC 下限
  - 充放电效率
  - 额定充放电功率
  - 不能同时充放电
- 输出容量/功率对以下指标的影响：
  - 年电费
  - 光伏自发自用率
  - 购电峰值
  - 年等效循环次数
- 增加工程经济性分析：
  - 储能投资成本
  - PCS 成本
  - 静态回收期
- 增加敏感性分析：
  - 储能成本
  - 电价差
  - 容量变化
  - 功率变化
- 给出三类候选方案：
  - 经济型
  - 均衡型
  - 鲁棒型

**注意：**  
D 不写“绝对最优方案”，只给不同目标下的候选配置。

**时间节点：**

- **周五晚上前**：接收 B/C 的最终数据，准备储能扫描模型。
- **周六中午前**：完成储能扫描初版。
- **周六下午前**：完成成本、回收期、循环次数和三类候选方案。
- **周日前**：提交任务五文字、图表和三类方案表给 A/E。
- **周一前**：根据 A 的统一口径修改推荐表述。

**交付结果：**

- 储能扫描结果 CSV/Excel
- 容量-功率-年电费图
- 容量-自发自用率图
- 购电峰值对比图
- 成本与回收期分析表
- 电池循环次数分析
- 敏感性分析图表
- 经济型/均衡型/鲁棒型方案表
- 任务五报告文字

## 成员 E：策略优化与电价情景负责人

**核心目标：**  
聚焦完成任务六，提供最终汇报的策略分析亮点。

**负责内容：**

- 对比三种储能充放电策略：
  - 基础光伏消纳策略
  - 上海峰谷电价策略
  - 日前预测策略
- 比较不同策略下的：
  - 年电费
  - 自发自用率
  - 峰值购电功率

  - 固定峰谷规则
  - 峰谷价差扩大
  - 上网电价变化
  - 峰段延长

**注意：**  
电价预测/电价情景只是轻量创新，不作为主线成败标准。如果复杂预测模型来不及做，则以电价情景分析作为最低交付。
