## 二、建立365天EV出行事件表

### 优化后的 EV 行为模型

模型目标：在固定通勤规律的基础上，引入**随机波动**、**周末差异**、**随机长途出行**和 **SOC 安全约束**，使 EV 负荷更接近真实家庭用车行为，同时保证后续三种充电策略具有统一、公平的比较基础

### 1. 基础参数

|       参数       |      数值      |
| :--------------: | :------------: |
|   EV 电池容量    |    62.5 kWh    |
|   单位里程耗电   |  0.15 kWh/km   |
|  工作日基准里程  |     35 km      |
|   周末基准里程   |     20 km      |
|   长途额外里程   | 80–200 km 随机 |
|     初始 SOC     |      90%       |
|   触发充电 SOC   |      40%       |
|  安全 SOC 下限   |      20%       |
|   日常目标 SOC   |      90%       |
| 长途预充目标 SOC |      100%      |

### 2. 每日出行规则

#### 工作日

周一到周五为通勤日

```text
实际里程 = 35 × (1 + 10%随机扰动)
限制范围：20–50 km
```

#### 普通周末

周六、周日为短途出行日

```text
实际里程 = 20 × (1 + 15%随机扰动)
限制范围：10–40 km
```

#### 随机长途日

每月随机选择 2 个周末日作为长途日

```text
长途日里程 = 普通周末里程 + 80–200 km随机额外里程
```

全年约 24 个长途日

### 3. SOC 行为规则

每日 SOC 更新顺序为：

```text
每日开始 SOC
→ 当天行驶耗电
→ 行驶后 SOC
→ 判断是否需要充电
→ 按充电策略补能
→ 充电后 SOC
```

### 4. 充电触发规则

#### 日常触发

```text
如果行驶后 SOC < 40%
则触发充电
目标 SOC = 90%
```

#### 周日补电

```text
每周日晚上强制补电至 90%
```

目的：保证周一通勤需求

#### 长途预充

```text
如果明天是长途日
则前一晚预充至 100%
```

目的：保证长途出行前电量充足

#### 安全 SOC 保护

```text
如果行驶后 SOC < 20%
则认为发生途中补能事件
并将行驶后 SOC 修正至 20%
```

目的：避免极端长途导致车辆电量低于安全边界

### 5. 策略比较原则

三种 EV 充电策略共用同一张 365 天 EV 出行事件表

```text
EV-S1 到家即充
EV-S2 谷时延迟充
EV-S3 光伏富余优先
```

即：

```text
同一辆车
同一年出行行为不变
只改变充电策略
```

这样可以避免随机出行差异影响策略评价，保证对比结果公平

### 6. 模型特点

这个版本相比中期模型的改进是：

```text
固定里程 → 随机里程
固定长途 → 随机长途日 + 随机长途里程
单一充电触发 → SOC阈值 + 周日补电 + 长途预充 + 安全SOC保护
单策略建模 → 统一出行事件表支撑三种策略公平对比
```

## 三、写SOC更新逻辑

### 当前 EV SOC 更新逻辑概述

SOC 更新逻辑基于之前生成的 EV 出行事件表和 5 分钟基础负荷时间序列。整个流程分为每日和每 5 分钟分辨率两层：

### 1. 每日 SOC 更新顺序

1. **每日初始 SOC**

   * 从前一天晚上 SOC 结束值或初始 SOC 90% 开始
2. **行驶耗电**

   * 每日行驶耗电由 EV_event_table 中 `drive_energy_kWh` 给出
   * 当日耗电按 5 分钟时间段均匀分配
   * SOC 更新公式：

     ```
     SOC(t+dt) = max(SOC(t) - drive_energy(t)/E_ev_bat, SOC_min)
     ```
3. **充电触发判断**

   * 条件：

     * 行驶后 SOC < 40%（日常触发）
     * 周日补电（补到 90%）
     * 长途预充（补到 100%）
4. **充电执行**

   * 依据策略计算充电功率：

     * EV-S1：到家即充，晚间 19:00 充至目标 SOC
     * EV-S2：谷时延迟，22:00 充至目标 SOC
     * EV-S3：光伏富余优先，白天光伏富余充电，夜间不足补充
   * SOC 增量：

     ```
     SOC(t+dt) = min(SOC(t) + charge_energy(t)/E_ev_bat, SOC_target)
     ```
5. **安全 SOC 约束**

   * SOC 不超过目标 SOC
   * SOC 不低于安全下限 20%
   * 超过安全下限时视为途中补能事件

### 2. 每 5 分钟分辨率逻辑

* 每 5 分钟计算 SOC 变化：

  1. 扣除当前时间段行驶耗电（均分当天耗电）
  2. 判断充电触发条件
  3. 若允许充电，按充电功率更新 SOC
* 充电功率限制：

  ```
  P_charge ≤ 充电桩额定功率 (7 kW)
  ```
* SOC 上下限约束：

  ```
  SOC_min ≤ SOC(t) ≤ SOC_target
  ```

### 3. 核心公式

```matlab
SOC(t+dt) = SOC(t) - drive_energy(t)/E_ev_bat + charge_energy(t)/E_ev_bat
SOC(t+dt) = min(SOC_target, max(SOC_min, SOC(t+dt)))
```

* `drive_energy(t)`：当前 5 分钟耗电
* `charge_energy(t)`：当前 5 分钟充电量，依据策略决定
* `SOC_min = 0.2`，`SOC_target = 0.9` 或 `1.0`（长途预充）

### 4. 策略共用出行表

* EV-S1、EV-S2、EV-S3 都使用同一个 `EV_event_table.csv`
* 保证策略对比公平，避免随机出行差异影响评估
* SOC 逻辑不变，只改变充电时段和充电方式

整体逻辑保证：

* 可复现性
* 公平的策略对比
* 安全 SOC 管控
* 支撑全年 5 分钟分辨率负荷和光伏数据

## 四、先实现EV-S1

### EV-S1 到家即充逻辑
**固定出行行为 + 晚上 19:00 到家即充 + 充电直到目标 SOC + 安全 SOC 保护 + 输出 SOC/EV功率/总负荷**

### 1. 出行行为

* 每天的 EV 行程来自 `EV_event_table.csv`
* 包含工作日通勤、周末短途和随机长途日
* 每日消耗电量：

  ```
  drive_energy_kWh = distance_km * 0.15 kWh/km
  ```
* SOC 更新顺序：

  ```
  每日开始 SOC
  → 扣除行驶耗电
  → 行驶后 SOC
  → 判断是否需要充电
  → 充电后 SOC
  ```

### 2. 充电触发条件

* **日常触发**：SOC < 40%
* **周日补电**：每周日晚上强制补到 90%
* **长途预充**：如果第二天是长途日，前一晚补充到 100%
* **安全 SOC**：行驶后 SOC 不低于 20%，否则视为途中补能

### 3. 充电执行

* **策略 EV-S1**：

  * 晚上 19:00 到家后开始充电
  * 充电功率上限 7 kW
  * 充电效率 0.9
  * 持续充电直到达到目标 SOC（90% 或长途预充100%）

### 4. SOC 更新公式

每 5 分钟时间步：

```
SOC(t+dt) = SOC(t) - drive_energy(t)/E_ev_bat + charge_energy(t)/E_ev_bat
SOC(t+dt) = min(SOC_target, max(SOC_min, SOC(t+dt)))
```

* `drive_energy(t)`：当前时间步消耗
* `charge_energy(t)`：当前时间步充入电池的能量，受充电桩功率限制和策略目标 SOC 控制

### 5. 总负荷计算

```
P_total(t) = P_load_base(t) + P_EV(t)
```

* `P_EV(t)`：当前时间步 EV 充电功率
* `P_load_base(t)`：基础负荷 5 分钟数据

### 6. 输出内容

* 年度 SOC 曲线（5 分钟分辨率）
* EV 每 5 分钟充电功率
* 加入 EV 后总负荷
* 晚高峰（18:00–22:00）峰值负荷
* 年充电总量、全年总负荷总量
* 途中补能次数统计

## 五、再复制扩展EV-S2

### EV-S2 谷时延迟充逻辑

EV-S2 的核心思想是：**出行行为不变，只改变充电时间**
它与 EV-S1 使用同一张 `EV_event_table.csv`，保证策略对比公平

### 1. 策略目标

EV-S2 不再采用 19:00 到家即充，而是将充电延迟到谷时段

```text
22:00 后开始充电
尽量避开 18:00–22:00 晚高峰
降低晚高峰负荷冲击
```

### 2. 出行行为

与 EV-S1 完全一致

```text
工作日随机通勤
周末随机短途
随机长途日
随机长途里程
```

每日 SOC 更新顺序仍然是：

```text
每日开始 SOC
→ 当天行驶耗电
→ 行驶后 SOC
→ 判断是否需要充电
→ 谷时段延迟充电
→ 充电后 SOC
```

### 3. 充电触发条件

触发条件与 EV-S1 一致

```text
行驶后 SOC < 40%
周日补电
长途预充
```

其中：

```text
日常目标 SOC = 90%
周日补电目标 SOC = 90%
长途预充目标 SOC = 100%
```

### 4. 充电执行规则

EV-S2 的区别在于充电开始时间

```text
若当天需要充电
不在 19:00 立即充
而是延迟到 22:00 后
以 7 kW 家用充电桩功率充电
直到达到目标 SOC
```

即：

```text
EV-S1：19:00 后允许充电
EV-S2：22:00 后允许充电
```

### 5. 预期效果

相比 EV-S1：

```text
晚高峰负荷应明显降低
全年最大负荷可能降低
EV年充电电量基本接近
SOC曲线整体相近
充电负荷从19:00后转移到22:00后
```

### 6. 适用意义

EV-S2 是一种典型的需求响应策略

```text
通过改变EV充电时段
将高峰用电转移到谷时段
缓解家庭晚高峰负荷压力
```

## 六、最后实现EV-S3

### EV-S3 光伏富余优先充电策略

### 1. 出行行为

* 与 EV-S1 和 EV-S2 共用同一张 `EV_event_table.csv`
* 每日行程、随机长途日、周末通勤与 SOC 参数不变
* 确保策略对比公平

### 2. 策略目标

EV-S3 的主要思想是：

```text
利用白天光伏富余电量优先为 EV 充电
若白天光伏不足，晚间谷时兜底补至目标SOC
```

目的：

* 提高光伏自发自用率
* 平滑晚高峰充电负荷
* 避免白天弃光

### 3. 充电触发条件

* 行驶后 SOC < 40%（日常触发）
* 周日补电至 90%
* 长途预充至 100%
* 安全 SOC 下限 20%

### 4. 充电执行规则

1. **白天光伏富余阶段**

```text
时间段：日间光伏出力 > 当前负荷
充电功率：min(P_ev_ch, PV富余功率)
持续充电至达到日常目标 SOC 或光伏不富余
```

2. **晚间谷时兜底**

```text
时间段：22:00 后
若当天未充满目标 SOC
继续充电至日常目标 SOC（周日补电或长途预充可能到100%）
充电功率上限：7 kW
```

### 5. SOC 更新顺序

每日 5 分钟分辨率更新：

```
每日开始 SOC
→ 当天行驶耗电
→ 行驶后 SOC
→ 白天光伏富余优先充电
→ 晚间谷时兜底充电
→ 更新 SOC
```

### 6. 总负荷计算

```
P_total(t) = P_load_base(t) + P_EV(t)
```

* P_EV(t)：光伏富余优先充电功率 + 谷时兜底充电功率
* P_load_base(t)：基础负荷

### 7. 预期效果

* 白天光伏能量被充分利用
* 晚高峰负荷显著低于 EV-S1 和 EV-S2
* 年总用电量与 EV-S1/EV-S2 相近，但峰值负荷更平滑
* SOC 范围仍维持在安全范围 20%-100%
